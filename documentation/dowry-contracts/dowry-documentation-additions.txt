Example transformation for gmn:P70_32_indicates_donor (UPDATED to include dowries):

INPUT (shortcut):
<dowry_contract> gmn:P70_32_indicates_donor <father_giacomo> .

OUTPUT (CIDOC-CRM compliant):
<dowry_contract> cidoc:P70_documents <acquisition1> .
<acquisition1> a cidoc:E8_Acquisition ;
               cidoc:P23_transferred_title_from <father_giacomo> .

Note: This property now works for both gmn:E31_7_Donation_Contract and 
gmn:E31_8_Dowry_Contract. The domain was expanded to include both contract types.

Example transformation for gmn:P70_22_indicates_receiving_party (UPDATED to include dowries):

INPUT (shortcut):
<dowry_contract> gmn:P70_22_indicates_receiving_party <bride_maria> .

OUTPUT (CIDOC-CRM compliant):
<dowry_contract> cidoc:P70_documents <acquisition1> .
<acquisition1> a cidoc:E8_Acquisition ;
               cidoc:P22_transferred_title_to <bride_maria> .

Note: This property works differently depending on the document type:
- For cessions (E31_4): P70_documents > E7_Activity > P14_carried_out_by
- For declarations (E31_5): P70_documents > E7_Activity > P01_has_domain
- For donations (E31_7): P70_documents > E8_Acquisition > P22_transferred_title_to
- For dowries (E31_8): P70_documents > E8_Acquisition > P22_transferred_title_to

PROPERTY DOMAIN DISTRIBUTION (UPDATED):

Properties shared across multiple contract types:
- P70.32 documents donor (E31_7 Donation, E31_8 Dowry)
- P70.22 indicates receiving party (E31_4 Cession, E31_5 Declaration, E31_7 Donation, E31_8 Dowry)
- P70.16 documents sale price amount (E31_2 Sales - but can be used by others for value)
- P70.17 documents sale price currency (E31_2 Sales - but can be used by others for value)

Properties specific to contract types:
- P70.1 documents seller (E31_2 Sales only)
- P70.2 documents buyer (E31_2 Sales only)
- P70.3 documents transfer of (E31_2 Sales only)
- P70.33 indicates object of donation (E31_7 Donation only)
- P70.34 indicates object of dowry (E31_8 Dowry only)

This distribution allows different contract types to share common properties while 
maintaining specific properties where the semantics differ.

COMPARISON OF TRANSFER CONTRACT TYPES:

| Property | Sales (E31_2) | Donation (E31_7) | Dowry (E31_8) |
|----------|---------------|------------------|---------------|
| Transferring Party | P70.1 (seller) | P70.32 (donor) | P70.32 (donor) |
| Receiving Party | P70.2 (buyer) | P70.22 (donee) | P70.22 (recipient) |
| Object | P70.3 (property) | P70.33 (donation) | P70.34 (dowry) |
| Payment | Required (P70.16-17) | Not expected | Often specified (P70.16-17) |
| Context | Commercial | Voluntary gift | Marriage arrangement |
| Event Type | E8_Acquisition | E8_Acquisition | E8_Acquisition |

IMPLEMENTATION NOTES:

1. Dowry contracts reuse P70.32 and P70.22 from other contract types, requiring updates 
   to the domain definitions of these properties in the ontology.

2. The transformation functions for P70.32 (donor) and P70.22 (receiving party) already 
   exist and will automatically handle dowry contracts once the ontology domains are updated.

3. A new transformation function is required only for P70.34 (object of dowry).

4. Like donations and sales, dowry contracts use E8_Acquisition to model the transfer, 
   maintaining consistency across property transfer contract types.

5. Dowry contracts can include monetary amounts (P70.16-17) to document the declared 
   value of the dowry, even though no purchase is involved.
